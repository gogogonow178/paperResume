## 1. 项目概述

- **产品定位**：一款零门槛、无后端、极简风格的在线简历编辑器。
- **核心价值**：所见即所得、隐私安全（本地存储）、Apple 风格的高级视觉体验。
- **目标用户**：追求高效、审美在线的求职者。
- **适用设备**：**仅限 PC/Mac 桌面端浏览器**（移动端访问显示提示页）。

------

## 2. 核心功能模块

### 2.1 整体布局

- **左侧（编辑区）**：宽度固定（约 400px），灰色背景。包含所有数据输入模块，支持折叠、拖拽排序。
- **右侧（预览区）**：宽度自适应，背景深灰。居中显示 A4 纸张样式的简历预览。
  - **多页逻辑**：当内容高度超过一页 A4 (297mm) 时，**自动生成第二页 A4**，两页之间有明显的视觉间隔（阴影分隔）。

### 2.2 数据存储与初始化

- **无登录系统**：所有数据实时保存在浏览器 `localStorage`。
- **默认状态**：
  - 首次访问或清空缓存后，加载一套完整的“张三”示例数据（让用户知道怎么填）。
  - 用户一旦开始编辑，覆盖默认数据。
- **一键清空**：顶部设置栏提供“重置/清空”按钮，点击后弹出二次确认，确认后清空所有输入项。

### 2.3 左侧编辑模块详解 (字段细化)

*模块支持拖拽排序（“基本信息”除外，固定在顶部）。*

#### A. 基本信息 (固定模块，不可移动/删除)

- **字段**：
  - **头像**：点击上传。
    - *逻辑*：前端自动压缩至 200KB 以内，裁剪为正方形，转 Base64 存储。
  - **姓名** (Text)
  - **求职意向** (Text)
  - **电话** (Tel)
  - **邮箱** (Email)
  - **所在城市** (Text)
  - **个人链接** (Text, 选填)：如 GitHub、个人博客、作品集链接。

#### B. 教育经历 (可排序列表)

- *每一项包含*：
  - **学校名称**
  - **学历/学位**
  - **就读时间** (开始年月 - 结束年月)
  - **主修专业**
  - **在校经历/描述** (多行文本，选填)

#### C. 工作经历 (可排序列表)

- *每一项包含*：
  - **公司名称**
  - **职位名称**
  - **在职时间** (开始年月 - 结束年月/至今)
  - **工作内容/业绩** (多行文本，**核心字段**)
    - *交互建议*：输入框支持自动换行，按回车自动生成项目符号（•），保持排版整洁。

#### D. 项目经历 (可排序列表)

- *每一项包含*：
  - **项目名称**
  - **担任角色**
  - **项目时间**
  - **项目链接** (选填)
  - **项目描述** (多行文本)

#### E. 专业技能 (自定义列表)

- *模式*：
  - 支持“项+程度”模式（如：Photoshop - 精通）
  - 或纯文本标签模式。
  - *建议采用*：**标题 + 内容块** 形式。例如标题填“设计软件”，内容填“PS, AI, Figma”。

#### F. 个人总结 (文本块)

- **字段**：纯多行文本区域，用于自我评价或致谢。

#### G. 自定义模块 (可添加多个)

- 允许用户增加一个通用模块，包含“模块标题”和“模块内容（富文本/多行文本）”，以应对“奖项荣誉”、“志愿者经历”等非标准需求。

## 3. 导出功能 (Export)

### 3.1 导出 PDF

- **逻辑**：
  - 将右侧预览区域（无论是一页还是多页）合并为一个 PDF 文件。
  - 保持 A4 尺寸比例，文字矢量化（可复制），不要变成模糊的图片。
  - 文件名默认格式：`姓名_求职意向_日期.pdf`。

### 3.2 导出图片

- **逻辑**：
  - **单页情况**：直接生成一张高分辨率 PNG 图片（至少 2倍图，保证打印清晰）。
  - **多页情况**：分别生成图片，打包成一个 ZIP 压缩包下载。

## 4. UI/UX 设计规范 (Apple Style)

### 4.1 视觉语言

- **字体**：
  - 英文/数字：San Francisco (Mac), Segoe UI (Win)。
  - 中文：PingFang SC (Mac), Microsoft YaHei (Win)。
- **色彩**：
  - 背景：`#F5F5F7` (Apple 浅灰)。
  - 卡片/纸张：`#FFFFFF` (纯白)。
  - 文字：`#1D1D1F` (深灰，避免纯黑)，辅助文字 `#86868B`。
  - 强调色/按钮：`#0071E3` (Apple 蓝)。
- **圆角与阴影**：
  - 模块圆角：12px - 18px。
  - 阴影：极轻的弥散阴影，营造悬浮感。

### 4.2 关键交互动画

- **左侧编辑区**：
  - **拖拽**：被拖拽的模块轻微放大并增加阴影，其余模块平滑移动让位（Transition 0.3s）。
  - **输入反馈**：右侧预览区实时更新，无刷新感。
- **移动端访问拦截**：
  - 当屏幕宽度小于 768px 时，全屏覆盖提示：“为了获得最佳的编辑体验，请使用电脑访问本网站。”

## 5. 技术可行性与注意事项 (Technical Notes)

- **技术栈建议**：React + Tailwind CSS (快速构建样式) + Zustand (轻量状态管理)。
- **PDF 生成库**：`html2canvas` + `jspdf` (最成熟的前端方案)。
- **本地存储限制**：
  - LocalStorage 上限约 5MB。
  - **必须执行**：图片上传时的 Canvas 压缩逻辑，确保图片转 Base64 后不超过 300KB，否则无法保存。
- **A4 分页计算**：
  - 这是一个技术难点。需要编写一个计算函数，监听右侧内容高度。
  - 当 `Height > 1123px` (A4 像素高度 @96dpi) 时，自动将溢出的 DOM 节点移动到新的 Page 容器中。
